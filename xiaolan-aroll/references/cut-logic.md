# Cut logic — /xiaolan-aroll 怎么决定剪什么

任务：把一条未剪辑的口播录像变成紧凑的成片。删三样东西，按*确定性*从高到低排：

1. **静音/停顿** — 永远可以放心收紧。
2. **被卡毙掉的 take** — 单独喊出的「卡 / 咔 / cut」标记它前面那条 take 是废的。
3. **重复的 retake** — 同一句说了两遍（喊没喊卡都算）；保留更干净的那条。

**参考脚本是 ground truth**。下面所有环节都用它来 (a) 比较之前先修 whisper 同音字，
(b) 两条 take 竞争时选保留哪条（最接近脚本的）。

---

## 1. 流水线数据流

```
input.mp4 ──ffmpeg──> audio.wav (16k mono)
audio.wav ──transcribe.py──> transcript.json   {duration, segments[{start,end,text,words[]}], words[]}
transcript WORD-GAPS + script.txt ──find-cuts.py──> edl.json {keep[], cuts[], stats, review[]} + edl.md
input.mp4 + edl.json ──apply-cuts.py──> <name>-edit.mp4      (select/aselect, CFR, synced)
```

## 2. 单元（takes）

`find-cuts.py` 用 **whisper 的 SEGMENTS 当单元**（`transcript.segments[]` 每项一个），
每个单元保留自己的 `words` 做字→时间映射。为什么用 segments 而不是按词间隙重新切分
（早期设计，2026-06-16 在一条真实录像上翻过车）：
- Whisper 本来就会把单独喊的**卡**切成独立 segment（cue 白捡），也绝不会把 `卡片`
  单独切成一个 segment（同音词天然安全）。
- 句子内部有犹豫停顿时，whisper 会把它保持为一个 segment（比如
  「我每次……发视频…赠品…拿走」），而按 `> 0.6 s` 间隙重切会把重说的那条打碎成片段
  （「我每次」|「发视频…」），然后把碎片误判成「重复」剪掉。用 segments 就避开了。

一个单元是 **cue**，当且仅当它归一化后的完整文本 ∈ `CUE_TOKENS`。每个非 cue 单元还带着
它最匹配的**脚本句索引**（`sidx`）+ **相似度**（`sr`）——只用于外科式前缀判定，
不用于重复检测（见 §4）。相似度用 `difflib.SequenceMatcher` 跑在去标点的字符序列上
（中文适用）。

## 3. 卡 cue

一个单元的完整文本是 cue token `{卡, 咔, 咔嚓, cut, カット}`（可配置 `CUE_TOKENS`）之一
时才算 cue。因为单元是整个 whisper segment，`卡片 / 卡顿 / 卡住`（卡融在长词里）永远
成不了 cue 单元——同音词靠构造就是安全的。

**Cue 驱动的 retake 解析**（小蓝的主机制——「比较卡前后」）：
- `pre` = cue 前最近的非 cue 单元；`post` = cue 后最近的非 cue 单元。
- 如果 `pre` 和 `post` 是**同一句**——互相相似度 ≥ `min(--sim, 0.40)`（cue 本身是强证据，
  所以门槛放宽）——就删废 take：
  - **外科式**（segment 内废起头）：如果只有 `pre` 的尾巴在重复 `post`，且尾巴前面的头
    是有脚本背书的完整分句（`len(prefix)≥4` 且 `best_script(prefix).sr ≥ 0.55`），
    只剪 `[重复尾巴起点 … cue.end]`，保留铺垫头。（处理「铺垫 + 假起头」粘在
    一个 segment 里的情况。）
  - 否则**整单元**剪：`[pre.start … cue.end]`。
- 如果它俩明显不是同一句：只剪掉 cue 这个词本身，pre 和 post **都保留**，记进
  `review[]`。cue 模糊时绝不悄悄删真句子。
- **局限：** 假起头如果自己是一个小碎 segment（「用」）而不是粘在铺垫上，`pre` 就只是
  那个碎片，跟 `post` 匹配不上 → 落进 `review[]`；手改 `edl.json` 收尾（把前一个 keep
  区间截到铺垫结束处）。2026-06-16 发生过一次，改一行 `keep[]` 就修好了。

## 4. 无 cue 的重复 retake

Cue 处理完后，按时间顺序走一遍剩下保留的非 cue 单元。每对相邻 `(prev, cur)`，若
**互相相似度高** `ratio(prev, cur) ≥ --sim`（默认 0.55），剪掉**前面**那条（`prev`）——
保留后面/更干净的 take（小蓝的「保第二遍」规则）。3 条以上的链逐对解析，
最后只有该句的最后一条 take 存活。

**关键护栏——看相似度，不看是否共享脚本句。** 早期版本把「同 `sidx`」当重复，
灾难性过剪：一句长脚本经常被**拆成好几个 segment 说出来**（比如脚本
「但长文……我高中……配图……」讲成 3 个 segment）。那是一句话的连续部分，不是重复——
它们共享 `sidx`，但互相相似度低。所以重复检测只用互相 `ratio`。
（脚本的 `sidx`/`sr` 仍用于 §3 的外科式前缀判定。）

## 5. 停顿修剪 — 静音优先 + 双档（小蓝，2026-07-05）

**为什么静音优先（2026-07-05 的教训）：** whisper 的词时间在停顿附近会撒谎——句尾词
往静音里拉伸，下一段的起点又提前伸进静音。在 32529 上实测：一个真实 1.10 s 的停顿
只显示为 0.22 s 的词间隙（「力」声称 14.52→15.24，而实际语音 14.51 就结束了）。
所以基于词间隙的修剪（2026-06-27 的设计）严重漏剪——大部分真停顿它根本看不见。
现在停顿是在音频上找的：`prep_audio()` 对 `--audio`（喂给 whisper 的同一个 16 k wav）
建 10 ms 的 RMS 包络，`quiet_runs()` 找低于阈值的连续段，
`quiet_th = max(2.5×noise-floor, 10 % of speech RMS)`（典型录音约 −45 dBFS——静音 +
轻呼吸可剪，任何带人声的都不行）。

**双档。** 每个静音段按它跟在哪个单元（whisper segment）后面分类：
- **句中**（同一单元内，或左侧单元不以句末标点结尾）：> `--min-pause`（0.15）的段
  压到 `--beat`（0.10 s）残留。小蓝的规矩：句子中间就不该停——语流保持完整。
- **句末**（左侧单元以 。！？!?… 结尾）：> `--boundary-pause`（0.25）的段压到
  `--boundary-beat`（0.20 s）——给句间的节奏留呼吸。

**词安全现在归「起音信任」（ONSET TRUST）规则管。** whisper 词仍然划定保留的 take，
剪切区间必须在下一个词的起音之前结束。但静音段**内部**声称的词起点，只有其后紧跟
真实能量才可信（检测窗口 clamp 在静音段内，防止上一个词的衰减冒充能量）。定在死寂里的
起点是时间伪影，直接跳过。残留 beat 永远在真实起音**之前**（剪切从 run-start + 0.02
开始），起音保有呼吸空间。2026-06-26 的 关注/如果 保护延续：whisper 转写出的语音在
真实起音处有能量，会终结静音段——静音段永远盖不住听得见的语音。只有比语音 RMS 的
~10 % 还轻的起音才可能被啃掉；如果某个重说的起音听着被削了，把那个窗口 VAD-off
重转写（§7 定稿步骤）——跟以前的修法一样。

**没传 `--audio` → 回退：** 词间隙修剪（双档，`place_cut()`），因上述原因漏剪停顿。
永远传 `--audio`。`--noise`/`silencedetect` 核心删除法已退役；`--noise` 接受但忽略。

**给已剪视频再收紧：** 在已剪好的视频上跑 `--silence-only --audio audio.wav`，
会干净地重转写并只跑停顿这一遍（不重判 take）。32529-edit.mp4 就是这样从 133.3 s →
110.0 s、零处 ≥ 0.3 s 静音残留的。

调参小抄：
- 想句中更紧 → 调低 `--beat`（0.06–0.08）；想松 → 调高（0.15）。
- 句间换气觉得赶 → 调高 `--boundary-beat`（0.25–0.30）。
- 小犹豫没剪干净 → 调低 `--min-pause`（0.12）；把真实的微停顿剪没了 → 调高。
- 轻声被啃 → 调高静音阈值护栏（`prep_audio`），或在 `edl.json` 里手修那个接缝。
- 真句子被当重复过剪 → 调高 `--sim`（0.7）并查 `review[]`。

## 6. 渲染（apply-cuts.py）

用一遍 **select/aselect**，视频和音频在完全相同的区间上剪（保持同步）：

```
[0:v]select='between(t,S1,E1)+between(t,S2,E2)+...',setpts=N/FRAME_RATE/TB[v];
[0:a]aselect='between(t,S1,E1)+between(t,S2,E2)+...',asetpts=N/SR/TB[a]
```

- filter 写进一个 **filter-script 文件**，通过 `-filter_complex_script` 传入——keep 列表
  再长也不会撑爆 Windows 的命令行长度限制。
- 输出**重编码为 CFR**：`-r <fps> -c:v libx264 -crf 18 -preset medium -pix_fmt yuv420p
  -c:a aac -b:a 192k`。CFR（`-r`）是必须的——VFR 输入 + 帧 select 会音画漂移。
- `--audio-only` 只渲染清理后的 `.wav`/`.m4a`（只用 aselect），快速试听用。
- select 式跳切不做 crossfade；音频接缝处可能有轻微咔哒声，粗剪够用了（后面在 CapCut
  精修）。想要零咔哒，换成逐段 `atrim + afade(6ms) + concat` 方案（filter 更重，
  区间一样）。

## 7. 坑（别再踩）

- **卡的同音词**（卡片/卡顿/卡住/卡路里）是内容——靠「单独出现 + 前后有停顿」这个判定
  保护。就算 whisper 把一个真·卡字词粘成了单词 segment，间隙判定仍然救得回来
  （它周围没有静音）。
- **Whisper vs 脚本：** 判重复用*归一化后对脚本比较*的文本，别用 whisper 原文。
  一个听错的词能让一条 take 看着不像它的 retake（→ 漏剪），也能让两句不同的话看着
  相似（→ 误剪）。两种僵局都由脚本打破。
- **不删口头语、不降噪** — 超出范围，会改变博主的声音。只删 停顿/废 take/重复。
- **音画同步：** `select` 和 `aselect` 的区间完全一致；输出永远强制 CFR `-r`；
  源是 VFR 时更是强制项。
- **留 padding + 别过度收紧：** 完全零 pad 的硬拼接会削爆破音、听着像机器人。
  默认 pad 0.10 s、min-pause 0.25 s（> 0.25 s 的静音剪到 ~0.20 s beat）是
  自然听感的起点。
- **有疑问绝不删：** 模糊的卡、低相似度的「重复」进 `review[]` 并保留。多保的在 CapCut
  里救得回来；多剪的 take 就永远丢了。
- **对脚本验证：** find-cuts 之后，每句脚本都应在存活的转写里恰好出现一次。有句子
  消失就是启发式过剪了——手修 `edl.json`。
- **修辞性重复不是 repeat（2026-07-09，第一个真实误报）：** 小蓝的脚本会把一句陈述
  复述成反问（「……它替我回答。问题来了，它怎么能替我回答呢？」）。互相 ratio 过了
  `--sim`，自动跑把陈述句剪了，删掉了一句真脚本。每一个 [repeat] 剪切都要对脚本核：
  被剪文本和保留文本若对到脚本里两个**都存在**的不同句子，恢复这刀。
- **清嗓（咳/咳咳）可能代替卡终结一条废 take（2026-07-09）：** cue 不会触发，
  而多 segment 的 take 又躲得过相邻对遍历。异常扫描的信号：一个纯 咳/咳咳 的 KEEP
  segment 标记着 take 接缝——把它跟 cue token 一起 grep，把咳嗽接缝连同它的废 take 簇
  一起手动剪掉。剪完残留的一个低于阈值（−35dB）的散音节，之后会被静音修剪吸收，
  可以接受。
- **手动剪完，渲染上补一遍 `--silence-only`（临时文件 → 验证 → 替换）：** 静音中段扣除
  的保守边距会在手动剪的接缝处留 0.4–0.6 s 的 beat——结构对了也超 §5 的句末档标准。
  silence-only 这一遍把每个接缝归一化；只有 时长（= keep 总和 ±0.2s）、silencedetect
  （零处 ≥0.3s）、接缝起音、字符 diff（0 删除）四项检查全过，才替换交付文件。
