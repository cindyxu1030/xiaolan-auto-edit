# 小蓝的 AI 自动剪辑技能（xiaolan-auto-edit）

中文 | [English](README.en.md)

我给我的 AI 剪辑师写的「员工手册」，现在开源给你。

两个技能（skill），组成一条口播视频的自动剪辑流水线：

| 阶段 | 技能 | 干什么 |
|---|---|---|
| 1 — 粗剪 | [`xiaolan-aroll`](xiaolan-aroll/SKILL.md) | 吃进没剪过的口播原片 + 参考文稿：去停顿、剪掉喊「卡」作废的口误条、重复口条只留最干净的一遍，渲染出音画同步的粗剪成片，附一份可复查的保留/删除记录（EDL）。 |
| 2 — 包装 | [`xiaolan-broll`](xiaolan-broll/SKILL.md) | 吃进整条口播视频：自动做品牌锁定的配图动画（b-roll）+ 逐字卡拉OK字幕，输出一层带透明通道的 ProRes MOV 叠加层，叠回你的高清原片上。金句才配巨字招牌，普通句子老实打字幕。 |

## 为什么叫「员工手册」

这两份 SKILL.md 里几十条带日期的规矩，全是我的 AI 剪辑师翻车挨骂之后沉淀下来的：

- 字幕要贴着出镜人的头顶，不许吊在半空（2026-06-17）
- 白色的字禁止使用——奶油底上直接隐身
- 红色一条视频最多出现三次，用多了廉价
- 开工前先量出镜人的头在画面里有多高，图案永远不压脸
- 画插画这步永远不许跳，交活前自己一帧一帧检查

在聊天里骂 AI，第二天他就忘了。写进手册里的规矩，他每次开工前必读，才认。

## 安装

把两个技能文件夹拷进你的 coding agent 的技能目录（以你所用 agent 的文档为准，常见如 `~/.codex/skills/` 或对应位置）：

```
<你的技能目录>/xiaolan-aroll/
<你的技能目录>/xiaolan-broll/
```

触发词写在每个技能 frontmatter 的 `description` 里，中英文都行（"剪我的口播" / "edit my a-roll"）。

## 依赖

- 一个支持 SKILL.md 技能格式的 coding agent（Codex CLI、智谱 GLM 系 agent 等）
- `ffmpeg`（切割 + 渲染）
- Python 3 + `faster-whisper`（转写）；`rembg`（绿幕抠像）
- HyperFrames CLI（`npx hyperframes`，渲染 b-roll 叠加层）

## 目录结构

每个技能都是标准三件套：

- `SKILL.md` — 入口（员工手册本体：frontmatter + 工作流 + 铁律）
- `references/` — 按需加载的详细算法和配方
- `assets/` — 技能要跑的 Python 脚本和模板

## 改成你自己的

规矩是我的审美，流水线是通用的。推荐的用法：

1. 先原样跑通一条视频
2. 每次验收不满意，别只在聊天里说——把那句话写成一条规矩，带上日期，加进 SKILL.md
3. 给你的 AI 员工建一个「错题本」文件，学到的坑都记进去
4. 素材库复用优先：生成过的绿幕素材归档，先翻库再画新的

错一次算他的，错两次算你的——谁让你没把规矩写下来。
